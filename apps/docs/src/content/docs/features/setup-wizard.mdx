The first-login setup wizard guides new Breeze administrators through essential platform configuration immediately after their initial login. It is a two-step interactive workflow that covers organization setup and device enrollment. The wizard appears when the API returns `requiresSetup: true` on the `/users/me` endpoint. Once finished, the user is redirected to the main dashboard and the wizard never appears again.

The wizard is implemented as a React client-side component rendered at the `/setup` route. All steps are individually skippable, and the entire wizard can be dismissed at any point.

---

## When the Wizard Appears

The setup wizard is triggered when the API's `userRequiresSetup` function returns `true`. This function checks three conditions — all must be met:

| Condition | Purpose |
|-----------|---------|
| `setupCompletedAt` is `null` | User has not completed or skipped the wizard |
| `passwordChangedAt` is `null` | User has not changed the default password |
| `email` is `admin@breeze.local` | Only the seeded admin account triggers the wizard |

The frontend checks `requiresSetup` from the `GET /users/me` response and redirects to `/setup`. The wizard component performs an additional server-side check via `GET /users/me` to confirm the flag before rendering.

<Aside type="tip">
  The wizard only appears for the seeded `admin@breeze.local` account before they complete setup or change their password. Users created through invitations or the user management API will never see the wizard.
</Aside>

---

## Setup Steps

The wizard presents two sequential steps. Each step has both a **Save & Continue** action and a **Skip** button.

<Steps>
1. **Organization** -- Create or configure your organization and site. This step sets up the multi-tenant hierarchy (partner, organization, site) that structures your Breeze deployment. You can name your organization and create your first site.

2. **Enroll Device** -- Enroll your first device into Breeze. The step provides enrollment instructions and an enrollment key so you can install the agent on your first endpoint and verify connectivity.
</Steps>

Clicking **Go to Dashboard** on the final step calls `POST /system/setup-complete`, which sets the `setupCompletedAt` timestamp on the user record, clears the localStorage progress key, and redirects to `/`.

---

## Prerequisites

The setup wizard requires the following to be in place before it can run:

- **API server running** -- The wizard communicates with the API for all data loading and mutations.
- **Redis available** -- Required for rate limiting.
- **Authentication** -- The user must be logged in. Unauthenticated visitors to `/setup` are redirected to `/login`.

<Aside type="caution">
  If the API is unreachable during a step, the wizard displays an error message but still allows the user to continue or skip.
</Aside>

## Skipping and Resuming

### Skipping Individual Steps

Every step has a **Skip** button that advances to the next step without making any API calls. Skipped steps are tracked internally. No data is modified when a step is skipped.

### Skipping the Entire Wizard

A **Skip Setup** link appears below the step card on the first step. Clicking it calls `POST /system/setup-complete` to mark setup as done, clears the localStorage progress key, and redirects to the dashboard. The wizard will not appear again.

### Resuming After Interruption

The wizard persists the current step index to `localStorage` under the key `breeze-setup-step`. If the browser is closed or the user navigates away, reopening `/setup` restores the last active step. This state is cleared when the wizard is completed or skipped.

<Aside>
  If the user logs out before completing the wizard, they will be redirected to `/setup` again on their next login as long as `requiresSetup` is still `true`. The localStorage key ensures they resume from where they left off.
</Aside>

## API Reference

The setup wizard uses the following API endpoints:

| Method | Path | Purpose |
|--------|------|---------|
| `GET` | `/users/me` | Check `requiresSetup` flag to determine if wizard should render |
| `POST` | `/system/setup-complete` | Mark setup as finished, sets `setupCompletedAt` on user record |

### `POST /system/setup-complete`

Sets `setupCompletedAt` and `updatedAt` timestamps on the authenticated user's record. Requires JWT authentication. Returns `{ success: true }` on success or `{ error: "Failed to complete setup" }` with a `500` status on failure.

## Troubleshooting

**The wizard does not appear after first login.**
The wizard activates when the API returns `requiresSetup: true` from `/users/me`. All three conditions must be met: `setupCompletedAt` is `null`, `passwordChangedAt` is `null`, and the account email is `admin@breeze.local`. Verify all three fields on the user record.

**The wizard keeps appearing after completion.**
The `POST /system/setup-complete` call may have failed. Check the API logs for errors on the `/system/setup-complete` endpoint. Verify that the database is writable and that the `setup_completed_at` column exists on the `users` table.

**Browser shows a loading spinner indefinitely.**
The wizard performs an auth check on mount via `GET /users/me`. If this request hangs (e.g., API server is down or CORS is misconfigured), the spinner will never resolve. Check that the API server is running and that CORS allows requests from the frontend origin. Open the browser developer console for network errors.
